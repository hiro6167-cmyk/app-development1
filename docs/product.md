# product.md — プロダクト要件定義書

## 1. プロダクト概要

### 1.1 プロダクト名
**PositiveVoice**（ポジティブボイス）

### 1.2 一行サマリー
日常の小さな幸せと理想の世界を共有する、承認欲求に縛られないポジティブSNS

### 1.3 エレベーターピッチ
**SNS疲れ**を感じている**中高生・大学生**のための
**PositiveVoice**は**ポジティブ特化型SNSアプリ**です。
**いいね機能があるSNS**とは異なり、**承認欲求に縛られず純粋にポジティブな気持ちを共有できる場**を提供します。

### 1.4 背景と動機
- **課題**: 既存SNSは「いいね」数に一喜一憂し、承認欲求に疲弊するユーザーが多い
- **機会**: Z世代を中心に「デジタルウェルビーイング」への関心が高まっている
- **差別化**: いいね機能を排除し、ポジティブな投稿のみに特化することで、健全なSNS体験を提供

## 4. ユーザーフロー

### 4.1 コアフロー（投稿閲覧）
```
アプリ起動 → ホーム画面（タイムライン）
         → タブ切り替え（今日あったいいこと / 世界にこうなってほしい）
         → 投稿タップ → 投稿詳細 → コメント閲覧/投稿
```

### 4.2 投稿フロー
```
ホーム画面 → [+]ボタン → 投稿作成画面
         → テキスト入力 → 画像追加（任意）
         → タイプ選択 → [投稿する] → ホーム画面に戻る
```

### 4.3 オンボーディングフロー
```
初回起動 → スプラッシュ → オンボーディング（3ページ）
       → 認証画面 → ログイン/新規登録
       → プロフィール設定 → ホーム画面
```

### 4.4 エラー・エッジケースフロー
- **オフライン時**: 「ネットワークに接続してください」表示 + リトライボタン
- **データ0件時**: 空状態イラスト + 「最初の投稿をしてみましょう」
- **認証エラー**: 再ログイン促進
